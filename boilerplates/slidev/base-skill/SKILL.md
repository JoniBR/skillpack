# slidev (skillpack base)

This project was generated by `skillpack scaffold slidev`. The boilerplate
is the official Slidev starter — Vite + Vue 3 + a single markdown file
(`slides.md`) — pre-wired so an agent can edit the deck immediately.

## What's already done for you

- Installed `@slidev/cli`, `@slidev/theme-default`, `@slidev/theme-seriph`,
  and `vue`. You **do not** need to run `pnpm create slidev` — the upstream
  Slidev skill mentions that as step one; skip it for this project.
- A 5-slide demo deck at `slides.md` (title, content, code, image,
  thank-you). Edit in place.
- npm scripts wired:

| Script   | Command         | What it does                                              |
| -------- | --------------- | --------------------------------------------------------- |
| `dev`    | `slidev`        | Dev server at `http://localhost:3030` with HMR            |
| `build`  | `slidev build`  | Build a static SPA into `./dist/`                         |
| `export` | `slidev export` | Export to PDF (needs `pnpm add -D playwright-chromium`)   |

## Project conventions

- **Entry:** `slides.md` is the deck. Slides are separated by `---`.
  The first frontmatter block at the top is the **headmatter** (deck-wide
  config: `theme`, `title`, `transition`, etc.). Subsequent `---`-delimited
  blocks inside the file are per-slide frontmatter.
- **Components:** custom Vue components go in `./components/*.vue` and are
  auto-imported by Slidev. Use them inline in `slides.md`.
- **Assets:** static images live in `./public/` and are referenced as
  `/foo.png`. Remote URLs are cached at build time.
- **Themes:** swap by changing `theme: default` to e.g. `theme: seriph` in
  the headmatter. Both `@slidev/theme-default` and `@slidev/theme-seriph`
  are pre-installed.
