# react (skillpack base)

This project was generated by `skillpack scaffold react …`. The boilerplate is
Vite + React 18 + TypeScript + Vitest, organised so that **skills** (Remotion,
Confetti, etc.) can layer cleanly onto a single `App.tsx`.

## Project conventions

- **Entry:** `src/main.tsx` mounts `<App />` from `src/App.tsx` into `#root`.
  Don't move these — `skillpack` skills reference them by path.
- **Component layout:** new top-level UI goes inside `<App />`, between the
  marker comments. Reusable components live under `src/components/`. Hooks
  live under `src/hooks/`. Tests sit next to source as `*.test.tsx`.
- **Markers:** `App.tsx` contains two skillpack markers:
  - `// @skillpack:imports` — append new imports above this line via
    `markers[].imports`, not by editing the file directly.
  - `{/* @skillpack:mount */}` — insert JSX siblings inside `<main>` via
    `markers[].insert`, in CLI argument order.
- **Imports:** prefer `import { Foo } from './foo.js'` (note `.js` extension)
  to match the TS bundler-resolution mode of this project.

## Testing

- Test runner: **Vitest** with `globals: true`, `environment: 'jsdom'`.
- DOM matchers from `@testing-library/jest-dom` are auto-imported via
  `vitest.setup.ts`.
- Use React Testing Library (`render`, `screen`, `fireEvent`, `userEvent`).
  Query by role/text — avoid `data-testid` unless nothing else works.

## Common patterns

**Adding a component:**

```tsx
// src/components/Greeting.tsx
import React from 'react';
export function Greeting({ name }: { name: string }): React.ReactElement {
  return <p>Hello, {name}.</p>;
}
```

Then mount it from `App.tsx` (or, if you're authoring a skill, declare a
marker insert in your skill's `manifest.json`).

**Adding a hook:**

```ts
// src/hooks/useDocumentTitle.ts
import { useEffect } from 'react';
export function useDocumentTitle(title: string): void {
  useEffect(() => {
    const prev = document.title;
    document.title = title;
    return () => { document.title = prev; };
  }, [title]);
}
```
